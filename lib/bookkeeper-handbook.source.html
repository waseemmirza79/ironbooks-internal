SNAP Basics

Purpose
Sign in, learn the layout, and understand how clients move through SNAP.
Who / When
Every bookkeeper · first login, then daily.

Logging in

  1. Go to app.ironbooks.com and enter your @ironbooks.com email.
  2. SNAP is passwordless — you'll get a magic link by email. Click it to sign in.
  3. If a link expires, just request a new one from the login screen.

Navigating

  1. The sidebar groups everything into Work (Today, Cleanup, Production, Clients, History), plus Tools and Admin for seniors.
  2. Today = daily transactions to clear · Clients = the full client list & status · History = past jobs.

Onboarding

Onboarding a New Client

Purpose
Take a newly-won sale all the way to "ready for cleanup."
Who / When
Lead/Admin runs the board; assigned bookkeeper does setup.

From GHL "Won" to the board

  1. When a deal is marked Won in GHL, a webhook creates a lead on the Onboarding board (New sale → In onboarding → Ready to onboard).
  2. If a sale is missing, click Sync to backfill from GHL (no duplicates).
  3. Assign an owner (bookkeeper) to the new client.

Setup checklist (the three pieces)

  1. Connect QuickBooks (see SOP 3) — nothing else can start without it.
  2. Onboarding form — when the client submits it, answers auto-populate their profile. Review them on the Profile tab.
  3. Onboarding call — get it booked; the card shows "Call not booked / attended." The recorded call later appears under Call recordings (SOP 13).

Complete the profile

  1. On the Profile tab, fill the editable Client details card: contact, legal/business name, corporate type, fiscal year-end.
  2. Set jurisdiction (country/province) — drives tax handling and the US-vs-CAD payment link.
  3. Complete operational fields (revenue range, payroll provider, prior bookkeeper, etc.) and Save.

Match to Double & invite to the portal

  1. Use Match Client to Double to link the QBO client to its Double HQ record (clear any pending_ placeholder).
  2. From Admin → Invite client, send the portal magic-link invite; use Resend/Revoke as needed.
📸

Connecting QuickBooks

Purpose
Authorize SNAP to read & write the client's QBO — the prerequisite for all cleanup and reporting.
Who / When
Assigned bookkeeper · during onboarding; reconnect if it ever expires.

Connect

  1. On the client's Overview, the QBO banner shows connection health. If amber ("not connected"), click Connect QuickBooks.
  2. Sign in to Intuit with the bookkeeper QBO account, choose the right company file, and click Connect.
  3. You'll land back in SNAP — refresh once if the banner doesn't turn green.

Reconnect & diagnose

  1. A red banner ("expired") means the refresh token aged out (~100 days) or the client revoked SNAP — click Reconnect and redo the flow.
  2. Empty P&L? Check the banner first: green=connected, red=reconnect, amber=connect. It's the single source of truth.
📸
Screenshot — QBO connection banner
The banner in its connected (green) and expired (red) states.

⚠️
Until QBO is connected/reconnected, all financial tabs are empty — don't mistake an expired token for "no data."

Chart of Accounts Cleanup

Purpose
Standardize the client's chart of accounts before categorizing anything.
Who / When
Assigned bookkeeper · first cleanup step after QBO connect.

Your options

ToolUse it for
Cleanup workflowThe step-by-step path from new client to clean books.
AI COA FixSnapshot → AI analysis → review → finalize. Pre-accepts ≥95%-confidence, low-risk changes; you review the rest.
COA EditorOne-off manual fixes — add/rename/inactivate accounts, drill into transactions.
Master COAThe standard chart of accounts to align every client toward.

Steps

  1. Open the client's COA and compare it to the Master COA.
  2. Run AI COA Fix to handle the obvious standardizations; review anything below 95% confidence.
  3. Use the COA Editor for any remaining one-off fixes.
  4. Goal: a clean, standardized COA you can categorize against.
📸

Categorizing Transactions (Reclassify)

Purpose
Categorize the client's transactions against the cleaned COA, with AI assistance.
Who / When
Assigned bookkeeper · after the COA is in good shape.

Run → review → execute

  1. Open Reclassify → New Reclassification Job; pick the client and scope.
  2. On the Review screen, accept good AI proposals and correct the rest. Nothing posts yet.
  3. For big jobs, categorization runs in chunks — use the Continue gate to resume; progress is saved.
  4. Resolve uncategorized / low-confidence items; when you genuinely can't tell, use Ask Client rather than guessing.
  5. Execute to write the approved categorizations to QBO. Failed items surface for retry.
📸
Screenshot — Reclass review
The review screen with proposed categories, confidence, and the Continue gate.
/reclass/[id]/review
⚠️
Execute writes to live books — finish the review before running it. A clean book has no "Uncategorized" bucket left.

Bank Rules

Purpose
Turn categorization decisions into reusable QBO bank-feed rules so future months auto-categorize.
Who / When
Assigned bookkeeper · after a reclass job, or as a standalone pass.

Steps

  1. Build rules from a reclass job ("From Reclassification") or via New Bank Rules Discovery, which groups history by vendor.
  2. Review each proposed vendor → category rule; keep recurring vendors, skip one-offs.
  3. Export to QuickBooks — rules write into the client's bank feed. The profile tracks active-rule count + last export.
📸

⚠️
Exported rules act on the live feed — an over-broad rule miscategorizes future transactions automatically.

Balance Sheet Cleanup

Purpose
Reconcile the balance sheet account-by-account, using the right tool for each problem.
Who / When
Assigned bookkeeper · the BS phase of cleanup.

Home base

  1. Work from the BS Cleanup Wizard (guided reconciliation), going account-by-account.
  2. Reach for a specialized tool when an account is messy (below), then return to the wizard.

Specialized tools

ToolFixes
A/R RecoveryMoney that arrived but was never applied to the right invoice.
Uncategorized Income RecoveryDeposits stuck in Uncategorized Income → match to open A/R → clearing JEs.
UF AuditOrphan Undeposited Funds, grouped by customer → balanced clearing JEs.
UF-AIUpload CSVs → AI cleanup plan; set each group's deposit date (= the QBO posting date); Ask Client per group.
Hardcore CleanupUpload CRM job report → detect duplicate invoices from a CRM migration → JE write-offs/voids.

Posting

  1. Each tool ends by posting JEs to QBO — review that they're balanced and correctly dated.
  2. SNAP guards against closed-period and duplicate posting, but you confirm correctness before posting.
📸

Follow the money: overstated A/R usually means cash landed in Uncategorized Income or Undeposited Funds. The three recovery tools overlap — use them together.

Stripe Reconciliation

Purpose
Match Stripe deposits to invoices, splitting out fees and sales tax.
Who / When
Assigned bookkeeper · clients who take payment via Stripe.

Steps

  1. Connect Stripe: send the client a Stripe Connect link to authorize. No Stripe? Mark it not required.
  2. Open Stripe Recon → Stripe AR Reconciliation; pick client + date range. SNAP pulls deposits and matches them to invoices, calculating fees + sales tax.
  3. Review each deposit-to-invoice match and the fee/tax split, then Execute to post to QBO.

Completing & Submitting Cleanup

Purpose
Wrap a cleanup, get it approved, and hand the client a report.
Who / When
Bookkeeper submits; Lead/Admin approves.

Steps

  1. When cleanup is done, mark it complete and submit for review — the client moves to Ready for review.
  2. A Lead/Admin approves it (or reopens it with notes if something's off).
  3. Generate and send the cleanup report PDF to the client.
  4. If the balance sheet isn't done but you want to move forward, defer BS — the client shows "BS owed" and stays flagged until it's cleaned.

Production

Production & Daily Reconciliation

Purpose
Promote a clean client to daily recon and work the queue each morning.
Who / When
Assigned bookkeeper · once a client is cleaned up, then daily.

Promote to production

  1. On the client's Overview, once COA + reclass + bank rules are done, the "Move to production" CTA appears.
  2. Promoting turns on the 3am daily recon cron: new transactions auto-categorize at ≥95% confidence; the rest go to Today.
  3. You can Pause / Unpause / Disable daily recon anytime (with a reason shown to the team).

Work the Today queue

  1. Open Today each morning. Items are grouped anomaly / review / auto.
  2. Confirm or correct categories and clear items.
  3. Handle client reclass requests (approve/decline) and send Ask Client requests for missing info.
📸

The production CTA on a client, and the Today daily-recon queue.

⚠️
Only promote clients with a working QBO connection and a finished cleanup — daily recon learns from the baseline you leave it.

Month-End Close

Purpose
Close each production client's books every month.
Who / When
Assigned bookkeeper · monthly, per production client.

Steps

  1. Track month-end status on the Clients manager dashboard — each production client shows its month-end state (done / in review / waiting on client).
  2. Run the close for the period: clear the queue, reconcile, and finish the month-end.
  3. Finish month-end and notify the client their books are ready.

Ongoing

Client Service

Purpose
Report, communicate, follow up on calls, and manage billing — the ongoing relationship.
Who / When
Assigned bookkeeper · ongoing.

Reporting & the portal

  1. Review the client's P&L (date ranges, drill-downs, full COA) and Balance Sheet / Bank Balances on their tabs.
  2. Publish the P&L / Balance Sheet to the client portal when ready; SNAP can generate an AI narrative for their dashboard.

Communication

  1. Message the client from the inline thread on their profile, or send a one-way notification ("Your P&L is ready") that lights up their portal badge.
  2. Inbound client messages surface in the Today inbox — reply promptly.

Call recordings & to-dos

  1. Each client's Call recordings show the AI summary, a cross-call overview, and action items.
  2. Work open to-dos from the Overview panel (they cross off in the call card too). Unlink any wrongly-matched call.

Billing

  1. The Billing tab shows the client's tier, subscription, and invoice history.
  2. Handle tier changes, add-on coaching calls, and cancel requests from here.

Daily Recon — Cleanup-Done to Monthly Statements

Purpose
Keep a client's books current automatically every day, so month-end is verify + send — not "do the books."
Who / When
Lisa (manager) · the day cleanup is signed off, then daily + at month-end.
💡
The model in one line: once cleanup is signed off, put the client in production → SNAP auto-categorizes their bank activity every morning (7am) → you only review what it flags → at month-end you run checks, approve, and statements go out automatically. (Daily recon currently auto-categorizes incoming bank transactions, mainly expenses. Revenue matching, receipt capture, and Plaid auto-reconciliation are on the roadmap — for now handle those at month-end as usual.)

Step 1 · Put the cleaned-up client into production (one-time, ~5 min)

  1. Sidebar → Production → Daily Recon. Find the client; if not already enrolled, enroll them (turns on daily recon).
  2. Dry-run them first (the per-client Dry-run button) — this previews what the engine would categorize without touching QuickBooks. Skim it: do the obvious vendors look right (Sherwin-Williams → Paint & Materials)? Did it correctly hold the big / odd ones?
  3. If it looks sane, you're done — they run automatically at 7am nightly. The client's profile pill should read "In production."
⚠️
If the dry-run miscategorizes a lot, don't enroll yet — flag it; the client's rules / COA likely need attention first.

Step 2 · Daily review (~5–10 min each morning)

  1. The 7am run has already posted the confident transactions. Open Today — production clients show "X to review."
  2. For each flagged item: accept or correct the account, then Approve. Leave "Teach rule" ON so that vendor auto-categorizes next time (the queue keeps shrinking). Use Reject if it shouldn't post, Ask Client if you need info, and bulk-approve for the obvious batch.
  3. Give anomalies (big charges, payroll, unfamiliar vendors) a closer look — they're held on purpose.
  4. Target: queue at zero (or only genuinely-blocked items) each day.

Step 3 · Through the month

  1. Daily recon handles new activity — you mostly just clear the daily queue.
  2. Watch for a QuickBooks disconnect (you'll get a reconnect prompt / QBO-health flag) so the feed keeps flowing.
  3. Gather anything you'll need for the close (e.g., bank statements) — request via the client profile when relevant.

Step 4 · Month-end: close + send statements

  1. Sidebar → Production → Production board. Pick the close period; each production client is a card.
  2. For each client: Run checks (uncategorized, undeposited funds, A/R…) → review the Statements preview → fix anything a check flags (it links you to the tool) → submit / complete.
  3. Sidebar → Production → Approvals — your queue of files awaiting review + statements awaiting approval. Review and Approve.
  4. Approving automatically publishes the statements to the client portal, emails the PDF statements + login instructions, and closes the QuickBooks period. The client is done for the month.

Where things live

  1. Enroll / dry-run / monitor: Production → Daily Recon (/admin/daily-recon)
  2. Daily flag review: Today (/today)
  3. Month-end close board: Production → Production board (/production)
  4. Your approval queue: Production → Approvals (/approvals)
💡
Troubleshooting. No daily activity for a production client → confirm they're enrolled and the nightly run fired (if it's empty for everyone, it's the cron — escalate); force a catch-up with "Run all now" on the Daily Recon page. Engine miscategorized a vendor → fix the account before approving (it teaches the corrected rule). Too many flags → keep approving with "Teach rule" on; within a couple of weeks the regulars are covered.
